## Export Features

### Supported Chart Types
- **Bar Charts** (Vertical Columns)
  - Grid lines with Y-axis value labels
  - Value labels above each bar
  - X-axis category labels
  - Direct color: Theme accent (#FC2779)

- **Line Charts**
  - Connected data points with trend lines
  - Grid overlay with value scaling
  - Min/max value legend
  - Color: Theme accent light (#FF6B9D)

- **Pie Charts** (Segmented Bar Representation)
  - Horizontal stacked segments proportional to values
  - Percentage labels on segments
  - Legend with category names and percentages
  - Multiple color coding: [#FC2779, #FF6B9D, #FFB0D1, #FF85B4, #FFB8D9, #FFC9DC]

- **Data Tables**
  - Alternating row colors for readability
  - Header row styled with theme accent
  - Automatic column width distribution
  - Up to 12 rows per slide

### PPTX Structure
1. **Title Slide**
   - "InsightAI Report" heading
   - Dataset name + analysis subtitle
   - Total chart count and generation date
   - Decorative accent bar at top

2. **Chart Slides** (One per chart)
   - Slide number indicator (top-right)
   - Chart title in accent color
   - Metadata: Chart type + data point count
   - Insight/description box (if provided)
   - Chart visualization (9" × 3.5" area)

3. **Final Slide**
   - "End of Report" closing message
   - Generation date
   - "Generated by InsightAI V3" footer

### Theme & Styling
- **Background**: Dark mode (#080B14)

- **Accent Colors**: Pink gradient (#FC2779 → #FFC9DC)
- **Text**: White (#FFFFFF) with secondary gray (#A0AEC0)

## API Endpoint

### POST `/api/export/pptx`

**Request Format**:
```json
{
  "charts": [
    {
      "title": "Chart Title",
      "chart_type": "bar|line|pie|table",
      "insight": "Optional description",
      "data": [
        { "name": "Category", "value": 100 },
        { "name": "Category2", "value": 80 }
      ]
    }
  ],
  "datasetName": "Analysis Name"
}
```

**Response**: 
- Binary PPTX file with filename format: `InsightAI-Report-{timestamp}.pptx`
- Content-Type: `application/vnd.openxmlformats-officedocument.presentationml.presentation`

## Usage in Dashboard

### Export Button Flow
1. User clicks "Export PowerPoint" button in dashboard
2. Frontend collects all charts from Zustand store
3. POST request sent to `/api/export/pptx` with chart data
4. Backend generates PPTX using PptxGenJS library
5. File auto-downloads to user's default download folder

### Integration Points
- **Frontend**: `src/components/ExportPowerPointButton.tsx` - Trigger button
- **State**: `src/lib/store.ts` - `currentDashboard.charts` array
- **Backend**: `server/routes/export-pptx.ts` - Generation logic

## Limitations & Notes

1. **Pie Charts**: Implemented as segmented bars (not traditional pie) for better PowerPoint scaling
2. **Max Charts per Export**: No hard limit, but recommend 20-30 for performance
3. **Data Truncation**: 
   - Labels limited to 20 characters
   - Max 12 rows for tables
   - Max 6 segments for pie charts
4. **Font Support**: Arial only (universal PowerPoint support)
